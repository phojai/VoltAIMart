    Under the hood

    Tech stack

    What VoltAIMart is actually built with — for reference when debugging, extending, or handing this off.

    Architecture

    ● Solid arrows follow a real request end-to-end (browser → API → store). ● Dashed arrows are optional, pluggable integrations — only called when a key/assistant is configured in Settings. Locally, one Express process serves both the API and the static frontend; on Vercel, apps/web deploys to the static CDN and apps/api runs as a serverless function (api/index.js), with Upstash Redis standing in for the local db.json file.

    Frontend

    🧱
    Vanilla HTML / CSS / JS
    No framework, no build step — apps/web/. Storefront, cart, login, dashboard, account pages.
    🔌
    Shared fetch client
    js/api.js — auth headers, token storage, every API call.

    Backend

    🟢
    Node.js 18+
    Runtime for the API and CLI tooling (reset script, etc.).
    🚂
    Express 4
    apps/api/server.js — routing, JSON body parsing, CORS.

    Shopping & orders

    🧾
    Checkout & guest checkout
    Shipping address + simulated UPI/card payment — checkout.html. Guests just supply a contact email; logged-in users can save the address.
    📦
    Stock & inventory
    Per-product stock, enforced and decremented server-side on order, restored on cancel — orders.routes.js.
    ⭐
    Reviews & ratings
    Customer star ratings + written reviews roll up into each product's average rating.
    🤍
    Wishlist
    Server-persisted per user — wishlist.routes.js, wishlist.html.
    📍
    Address book
    Saved addresses reused at checkout — addresses.routes.js.
    🚚
    Order lifecycle
    Status timeline (processing → shipped → delivered) plus customer-initiated cancellation and public guest order tracking — track.html.
    📧
    Simulated notifications
    Order confirmed/shipped/delivered/cancelled "emails" recorded and shown in an account-page inbox — lib/notify.js is the single integration point for a real provider (SES/SendGrid/Resend) later.

    Auth & security

    🔑
    JWT (jsonwebtoken)
    7-day tokens; role checks (admin / agent / customer) enforced server-side.
    🔒
    bcryptjs
    Password hashing for every account.
    🛡️
    AES-256-GCM
    Encrypts stored LLM/search API keys at rest — lib/secretCrypto.js.

    Data & storage

    🗄️
    JSON-blob datastore
    Local dev: one file at apps/api/data/db.json.
    ☁️
    Upstash Redis
    Swapped in automatically on Vercel's read-only filesystem via @upstash/redis.
    🆔
    nanoid
    IDs for users, products, and orders.

    AI & voice

    🧠
    Anthropic / OpenAI / Gemini
    Pluggable per admin choice — tool-calling loop in lib/llm.js, raw fetch, no vendor SDKs.
    🌐
    Tavily
    Optional live web search tool for the chat assistant.
    🎙️
    Vapi Web SDK
    Optional live voice agent for the homepage mic button.
    🗣️
    Browser Speech API
    Free "simulated" voice fallback — zero setup, no API key needed.

    Deployment

    ▲
    Vercel
    Static CDN for apps/web + a serverless function at api/index.js.
    📦
    npm workspaces
    Monorepo: apps/web (frontend) + apps/api (backend).
